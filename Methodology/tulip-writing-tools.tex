\begin{slide}[toc=,bm=]{GitFlow Branches}
\begin{itemize}
	\item Main branches
	\begin{itemize}
		\item \textcolor{orange}{\textit{Master} branch} should be treated as the main flow of your work, 
		and it should be always production ready, 
		and in a ready to submit state.
		\item \textcolor{orange}{\textit{Develop} branch} is parallel to the master branch, 
		it is the main branch where the source code of HEAD always reflects a state with the latest 
		delivered development changes for the next release.
	\end{itemize}
	\item Assist branches	
	\begin{itemize}
		\item \textcolor{orange}{\textit{Feature} branches}: branch off from and merge back into develop branch. 
		They are used to develop new subsections (features) for the paper.
		\item \textcolor{orange}{\textit{Release} branches}: branch off from develop and merge 
		back into develop and master. 
		When the state of develop is ready for a submission, 
		we branch off and give the release branch a name reflecting the submission, 
		such as IJCAI2018.
		\item \textcolor{orange}{\textit{Hotfix} branches}: branch off from master and merge back into develop 
		and master. 
		It is typically used to fix those identified issues existing in all existing and working branches. 
		So we use it when revising based on the review comments of the submission.
	\end{itemize}
\end{itemize}
\end{slide}

\begin{slide}[toc=,bm=]{Rules}
	\begin{enumerate}
		\item Put only those files under version control that are directly modified by the user.
		\item Verify that your code can be compiled
		flawlessly before committing your modifications to the repository.
		\item Use compare tools to critically review your modifications before committing 
		them to the repository.
		\item Add a meaningful and descriptive comment when committing your modifications to the repository.
		\item Do not add big files into the repository unless it is really necessary.
	\end{enumerate}
\end{slide}

\begin{slide}[toc=,bm=,method=direct]{Rules for Template}
	To use these templates in collaborative writing, 
	there are some rules you need to follow. \\
	\begin{enumerate}
		\item Avoid ineffective modifications. 
		\item Do NOT change line breaks without good reasons.
		\item Turn off automatic line wrapping of your Latex editor.
		\item Start each new sentence in a new line.
		\item Split long sentences into several lines so that each line has at 
		most 80 characters.
		\item For all the marginal comments by other co-authors, 
		you MUST respond to them using a responsive marginal comment.
		\item If your comment has been properly addressed by co-authors, 
		you MUST remove or comment out both the original and 
		the response marginal comments.
		\begin{lstlisting}[language=TeX]
			\gangli{What is wrong here?}
			\qwu{This is my response ...}
		\end{lstlisting}
		\item You can use something similar to the following code to 
		indicate where you are updating up to.
		\begin{lstlisting} [language=TeX]
			\gliMarker %TODO: YourName up to here!
		\end{lstlisting}
	\end{enumerate}
\end{slide}

\begin{slide}[toc=,bm=]{Rules for \LaTeX + Git Workflow}
	\begin{enumerate}
		\item Frequent commits of minor changes
		\item Use the \textit{git} client for copying, 
		moving, 
		or renaming files and folders that are under revision control.
		\item Use \textit{git-flow} scheme to create \textit{feature}, 
		\textit{branch}, 
		\textit{release} and \textit{version} of your paper. 
		Be really careful when you are touching develop or master branch.
		\begin{itemize}
			\item \textit{master} branch represents the life of the public document, 
			which should always be ready for releasing to others;
			\item \textit{develop} branch accumulates all finished features and keeps the 
			life-cycle of all development before release.
			\item \textit{feature} branch for major items of work, 
			for example, 
			I am adding the section of experiment etc. 
			If you are quite sure that it can be done without affecting other 
			collaborating author’s work-flow, you can directly work on the \textit{develop} branch.
		\end{itemize}
	\end{enumerate}
	
\end{slide}
